# Hidden Rabbit: Статус Изменений (на 2026-04-07)

## 1) Что уже сделано

### 1.1 Базовые крупные доработки (по истории `origin/main`)
- Добавлена гибридная каскадная модель `Xray + Hysteria 2`.
- Добавлен smoke-check гибридного sidecar.
- Улучшена стабильность автонастройки (асинхронный setup, доработки SSH/credential path).
- Усилена установка Xray (ретраи, зеркала).
- Исправлен HTTP->HTTPS поток в режиме Greenlock: теперь жесткий `308`, а не meta-refresh.
- Усилен Xray user reconcile и fail-closed обработка agent runtime ошибок.

Ключевые свежие коммиты в `main`:
- `0a4ade0` — hard `308` redirect в Greenlock.
- `11b85f2` — fail-closed на `/sync` и расширение reconcile coverage.
- `9c48209` — post-setup reconcile/sync усиление.
- `28186dc` — фиксы Xray setup finalization и deploy-race в hybrid cascade.
- `8f40335` — выравнивание статусов bridge-ноды в UI, no-store для подписки, исключение `bridge/relay` из обычного Xray user-reconcile.

### 1.2 Что дополнительно подготовлено в рабочем дереве (еще не зафиксировано отдельным коммитом)
- Для Xray setup через API/MCP включен путь через `setupXrayNodeWithAgent` (а не чистый `setupXrayNode`), чтобы не получать «успешный setup без агента».
- Для Xray bridge сохранен отдельный режим `exitOnly`.
- Усилена установка `cc-agent`: зеркала, ретраи, строгая проверка `systemctl is-active`.
- Setup Xray сделан stricter по агенту (чтобы не помечать ноду готовой при нерабочем agent runtime).

## 2) Что подтверждено как рабочее

Проверено QA в тестовом контуре:
- `http://vpn1.hiddenrabbit.net.ru/` отдает `308 Permanent Redirect` на HTTPS.
- Single-node Xray setup: `xray.service` active, конфиг на месте, listener открыт.
- Single-node Hysteria setup (Relay1/Relay2): `hysteria-server` active, `xray-cascade` active, UDP/TCP listeners соответствуют роли.
- Каскад `Portal (Xray) -> Relay1 (Hy2) -> Relay2 (Hy2) -> Bridge (Xray)` поднимался end-to-end, все 3 ссылки были `online`.
- Подписка пользователя отдавалась `200` и содержала `vless://`.

Дополнительно подтверждено на живом стенде:
- `Happ` успешно подключается по заново добавленной подписке.
- Трафик реально проходит через все 3 сервера в цепочке `Xray -> Hysteria -> Xray`.
- `Xray-Bridge` отображается как `online` не только на карте, но и в списке нод и на dashboard.
- Ошибка `TLS handshake` воспроизводилась со старым клиентским состоянием/устаревшей подпиской, а не из-за нерабочего каскада.

Итоговый статус этапа:
- `P0-functional milestone`: подтвержден на рабочем стенде.
- `P0-release milestone`: еще не закрыт до повторного прогона на полностью чистых серверах.

## 3) Что еще не закрыто / риски

Целевые findings по аудиту:
- `P0`: риск потери/сброса agent-параметров при сохранении ноды из панели (нужна финальная верификация merge-поведения xray-полей).
- `P0`: ложный `online` при проблемах runtime sync в ветках, где ошибки не поднимаются до статуса задачи.
- `P1`: post-setup sync/redeploy местами fire-and-forget без строгого влияния на финальный setup state.
- `P2`: гонки reconcile при параллельных user-операциях (enable/disable/group change).

Что уже снято из P0-рисков на этом этапе:
- рассинхрон статусов bridge-ноды между картой, списком нод и dashboard;
- повторное добавление обычных user UUID на `bridge/relay` Xray-узлы;
- повторное использование клиентом закешированной подписки без re-fetch после смены UUID/маршрута.

Практические симптомы, которые уже встречались:
- Пустой `clients[]` в Xray runtime при формально успешном setup.
- Ошибки вида `invalid request user id`.
- Плавающие сетевые падения на скачивании Xray/agent бинарей.

## 4) Что сейчас точно надо прогнать на «чистом» контуре

1. Чистая установка панели + создание админа.
2. Автонастройка single Xray node.
3. Автонастройка single Hysteria node.
4. Полная цепочка `Xray -> Hy2 -> Hy2 -> Xray`.
5. Проверка link health (3/3 online).
6. Проверка подписки (код ответа, тип контента, наличие `vless://`).
7. Проверка повторного импорта подписки в клиент (`Happ`) после чистой установки.
8. Негативный тест: имитация проблем `cc-agent` и проверка, что setup корректно падает (без ложного success).

После успешного clean-run следующий этап:
1. зафиксировать `P0-release milestone`;
2. открыть `Sing-box PoC` под feature-flag;
3. сравнить стабильность/ресурсы/поведение каскада против текущего Xray+Hysteria runtime.
