## 3) Что еще не закрыто / риски

Целевые findings по аудиту:

Что уже снято из P0-рисков на этом этапе:
- рассинхрон статусов bridge-ноды между картой, списком нод и dashboard;
- повторное добавление обычных user UUID на `bridge/relay` Xray-узлы;
- повторное использование клиентом закешированной подписки без re-fetch после смены UUID/маршрута.

Практические симптомы, которые уже встречались:
- Пустой `clients[]` в Xray runtime при формально успешном setup.
- Ошибки вида `invalid request user id`.

## Update: 2026-04-08 — same-host panel+node preflight fix

Зафиксирован и устранён кодовый дефект same-host детекции на сервере, где панель и нода живут вместе.

Что ломалось:
- `isSameVpsAsPanel()` учитывал только `node.domain === PANEL_DOMAIN`, localhost и `PANEL_IP` env;
- в реальном same-host сценарии `node.ip` совпадал с IP панели, но DNS/IP match не проверялся;
- из-за этого `Xray`-нода на том же сервере пыталась занять `443` рядом с панелью и уходила в restart-loop;
- UI при этом показывал ложный `success`, а агент переходил в `unknown`.

Что сделано:
- same-host детекция стала async и теперь сравнивает `node.ip` с реально резолвящимся IP панели;
- `setupXrayNode()` теперь честно падает до старта сервиса, если `443/80` уже заняты панелью на том же сервере;

  - `masquerade.listenHTTP=80`
  - `masquerade.listenHTTPS=443`

Практический вывод:
